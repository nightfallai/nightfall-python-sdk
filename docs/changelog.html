Changelog¶

Here you can see the full list of changes between each Nightfall release.

Version 0.5.0¶

UNRELEASED

  • Publish Sphinx docs to GitHub pages

  • Split up unit and integration tests, use mocking for unit tests

  • Update scan and chunking method to scan strings from a dict

  • Stop attempting to split strings and instead raise an exception

  • Redefine MAX_PAYLOAD_SIZE constant to be actual max of 500_000 bytes

Warning

Scanning dicts instead of strings is a breaking change. Previous versions of the SDK accepted a list of strings for scanning. We now require users to pass in a list of dicts instead. nightfall.scan(['string']) should now be nightfall.scan([{'id': 'string'}]). This allows you to keep track of the reference of where the string came from for further processing.

Version 0.4.0¶

Released on June 20, 2021

  • Add debug logs to Nightfall module

  • Change primary class name from Api to Nightfall to make things a bit more clear when this library is used in other programs and allow from nightfall import Nightfall

Warning

This is a breaking change, since the previous version of this SDK imported Nightfall using from nightfall.api import Api

Version 0.3.0¶

Released on June 13, 2021

  • Implement basic chunking algorithm to split payloads per the API limts.
